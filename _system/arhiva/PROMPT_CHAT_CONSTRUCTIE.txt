
**STEP 1 - VERSION CHECK OBLIGATORIU (BLOCANT) [V27]:**

Prima linie a output-ului scriptului TREBUIE sa fie:

```

```

unde V27.X este versiunea curenta (V27.1 sau mai mare).

**Daca vezi:**
- `[pre_generation_check V27.X - schema REAL]` -> OK, treci la STEP 2
- `[pre_generation_check V26.X - schema INVENTED]` sau orice versiune
  mai mica -> **OPRESTE-TE IMEDIAT** si raspunde ARHITECT:
  
  > "Scriptul pre_generation_check.py atasat in OUT-V{NN}.zip este
  > versiunea VECHE buggy (cauta campuri inventate cover_label,

  > scriptul reparat cu schema reala (nume_hero_caps_rand1,
  > localStorage_studiu, localStorage_video)."
  
- Lipsa prima linie cu `[pre_generation_check ...]` -> scriptul e
  VECHE, fara self-check. Cere reatașare arhivei.

**NU PROCEDA LA GENERARE DACA SCRIPT VERSION != V27.X.** Versiunea
veche buggy genereaza fals pozitiv la L142.

**STEP 2 - INTERPRETARE EXIT CODE:**

- Exit 0 -> SPEC C{NN} INGHETAT + IDENTITATE POPULATA + FENOMENE ALINIAT.
  Treci la REGULA 1 si urmatoarele.
- Exit 1 -> Unul din cele 3 check-uri FAIL. **OPRESTE-TE IMEDIAT.**
  Scriptul a afisat deja mesajul complet pentru ARHITECT. NU adauga
  nimic peste mesajul scriptului. Asteapta decizia ARHITECT.
- Exit 2 -> Eroare SISTEM_TRAINITY.md lipsa/corupta. Raporteaza si
  cere ajutorul ARHITECT.

**NU PROCEDA LA GENERARE DACA EXIT CODE != 0.** Improvizarea SPEC in
chat CONSTRUCTIE genereaza inconsistente pedagogice intre constructii
si strica unitatea sistemului Trainity.

Exceptie: daca Bogdan scrie EXPLICIT "definim SPEC C{NN} acum in acest
chat", treci la modul SPEC FREEZING (intrebari structurate de inghetare
una cate una in format grila R-V03.56), apoi la generare dupa ce SPEC
e complet inghetat in SISTEM_TRAINITY.md.

**FORMAT OBLIGATORIU SPEC FREEZING (R-V03.56):**

Pentru fiecare element (INTRIGA, MIZA, MANTRA, MOTTO, 18 step-titles,
2 prompturi, 8 final-labels), motorul foloseste **format grila cu
3 variante creative**:

```
Intrebarea N din 9: [ELEMENT]

[Scurta explicatie ce rol are elementul]

La C04 [ELEMENT] era: "[exemplu inghetat]"

Pentru C{NN} ({nume_constructie}), iata 3 propuneri creative:

A. "[varianta 1]"
   [scurta nota pedagogica]

B. "[varianta 2]"
   [scurta nota concreta]

C. "[varianta 3]"
   [scurta nota poetica]

Alege A, B sau C. Daca niciuna nu te convinge, scrie "alte 3".
```

Cele 3 variante TREBUIE sa fie:
- Creative reale, NU placeholders
- Brand Trainity (zero em-dash, vocabular permis)
- Diferite ca abordare (pedagogic / concret / poetic)

ARHITECT poate:
- Alege A, B sau C direct
- Scrie "alte 3" -> motor genereaza 3 noi variante
- Combina: "A dar finalul din B" -> motor reformuleaza
- Edita: "B dar inlocuieste X cu Y" -> motor accepta

Motorul NU intreaba "care e INTRIGA TA?" - asta inverseaza rolurile.
Motorul propune, ARHITECT alege. R-V03.56 obligatoriu.
